🧭 心灵共鸣舱 · 配置与运行指南
这份指南将帮助你从零开始，在本地电脑上完整运行“心灵共鸣舱”——一个具备 AI 文本对话、图片理解与情绪回应能力的心理疗愈助手。

一、准备工作：安装基础环境
1. 安装 Node.js
访问 Node.js 官网，下载 LTS 版本（长期支持版）。
双击安装包，一路点击“Next”完成安装。
安装完成后，打开 命令提示符（Windows）或 终端（macOS/Linux），输入以下命令验证：
node -v
npm -v
如果正常显示版本号（例如 v20.x.x 和 10.x.x），说明安装成功。

2. 准备项目文件夹
将你收到的项目压缩包（或文件夹）解压到一个纯英文路径的目录中，例如：
D:/Program/
⚠️ 注意：路径中不要包含中文、空格或特殊符号，否则可能导致后续安装失败。
解压后，对于核心ai页面，项目文件夹（心灵共鸣舱/）内应包含以下文件：
./Pages/AI/
├── AI.html                 # ai前端页面
├── server.js               # 后端服务
├── package.json            # 依赖清单
├── package-lock.json       # 依赖版本锁定
├── .env                    # 环境变量模板（如果有）
└── (其他资源文件)
重要：如果压缩包内包含 node_modules 文件夹，请直接将其删除。我们必须在新电脑上重新安装依赖。

二、获取必需的 API 密钥
本项目使用了两个 AI 服务商，需要分别申请并获取密钥。

1. 豆包（火山引擎）- 负责文本对话
用于：AI 文字聊天、生成动态建议问题
访问 火山方舟控制台，注册/登录账号。
进入“API Key 管理”，点击“创建 API Key”，复制生成的密钥（格式为长串字符）。
进入“在线推理”，创建两个模型接入点（Endpoint）：
选择模型 Doubao-seed-1.6-lite 或 Doubao-pro-32k，创建后记录生成的 接入点 ID（格式如 ep-2025xxxxxx-xxxxx）。
将获取的 API Key 和 接入点 ID 暂存到记事本，稍后会用到。

2. 通义千问（阿里云百炼）- 负责图片理解
用于：识别用户上传的图片，并给出温柔的情绪回应
访问 阿里云百炼控制台，登录阿里云账号（需实名认证）。
在左侧菜单点击“API-KEY 管理”，点击“创建 API-KEY”。
复制生成的密钥（以 sk- 开头）。
同样暂存到记事本。

三、配置项目环境变量
在AI项目根目录（AI/）下，新建一个文件，命名为 .env（注意：文件名只有后缀，没有前缀）。
用记事本或 VS Code 打开该文件，填入以下内容，并将 = 后面的值替换为你自己的真实密钥：

# 豆包（火山引擎）文本模型
DOUBAO_API_KEY=你的豆包API_Key
DOUBAO_TEXT_MODEL_ID=你的文本模型接入点ID

# 通义千问（阿里云百炼）视觉模型
QWEN_API_KEY=你的阿里云百炼API_Key
保存文件。

🔒 安全提醒：.env 文件包含敏感密钥，请勿上传到公开仓库或分享给他人。

四、安装项目依赖
打开终端（命令提示符 / PowerShell / Terminal），通过 cd 命令进入项目根目录。例如：
cd D:\mind-cabin
执行以下命令，安装所有必需的依赖包：
npm install
等待安装完成，你会看到类似 added 123 packages in 10s 的提示。此时项目根目录下会自动生成 node_modules 文件夹。

五、启动后端服务
在终端中（确保仍在项目根目录），运行：
node server.js
如果一切正常，终端会输出：
✅ 多模态代理服务运行在 http://localhost:3001
注意：请保持此终端窗口一直运行，不要关闭。如果关闭，后端服务也会停止。

六、启动前端页面
在项目文件夹中找到 Index.html 文件。
双击用浏览器打开即可（推荐 Chrome 或 Edge）。

七、常见问题排查
找到ai对话页面后无法发送消息	    后端服务未启动	                  确认终端中已运行 node server.js 且未关闭
发送消息后提示“AI文本服务出错”	豆包 API 密钥无效或余额不足	      检查 .env 文件中的 DOUBAO_API_KEY 和 DOUBAO_TEXT_MODEL_ID 是否正确；登录火山方舟确认账户余额
上传图片后提示“AI视觉服务出错”	通义千问 API 密钥无效	              检查 .env 文件中的 QWEN_API_KEY 是否正确；确认阿里云百炼账户已开通模型服务
npm install 失败	            网络问题或 Node 版本过低	          尝试切换网络，或使用 npm config set registry https://registry.npmmirror.com 切换为国内镜像；确保 Node.js 版本 ≥ v18
node server.js 报错 MODULE_NOT_FOUND	依赖未正确安装	          删除 node_modules 文件夹和 package-lock.json，重新执行 npm install
